# Security Policy

*[Version française](SECURITY.fr.md)*

## Supported Versions

Dolispace is developed on a single `main` branch. Only the latest released version
receives security updates — please make sure you are running the most recent tag
before reporting an issue.

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public GitHub issue**.

Report it privately by email to **info@siladel.fr**, including:

- A description of the vulnerability and its potential impact
- Steps to reproduce it
- The affected version/commit

We will acknowledge your report as soon as possible and keep you informed of the
progress toward a fix. Please allow reasonable time for a patch to be released
before any public disclosure.

## Scope

This policy covers the Dolispace application code in this repository. It does not
cover the security of a self-hosted Dolibarr instance itself, third-party
dependencies (report those upstream), or the hosting infrastructure of a given
deployment.
